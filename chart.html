v1.3.4 Session Notes — 2026-04-21

Ask Goose grading: model upgrade to qwen2.5:7b arch
Swapped Ollama model from qwen2.5:1.5b to qwen2.5:7b-instruct-q4_K_M (~Haiku-tier reasoning) for more reliable rubric scoring. 14B variant was pulled but too slow on this CPU (>20min/inference at CPUQuota=250%). RAM upgrade enabled the jump — not CPU — so 7B is the quality ceiling on this machine. Grading takes ~7-8 min/session now vs ~30s on 1.5B.
Cron spread: */15 one-at-a-time arch
Replaced 0 3 * * * daily batch-of-5 with */15 * * * * one-session-per-fire. Script runs with --once flag + lockfile at /tmp/grade-ask-goose.lock, falls through when thermal-gate detects CPU >75°C. Natural ~7-on / 7-off cadence avoids sustained thermal load while catching up ~55 backlogged sessions overnight.
Backfill-aware session picker feature
New fetch_next_ungraded_by_current_model() queries chat_grades filtered by the current OLLAMA_MODEL constant, builds a set of already-graded session IDs, then returns the oldest chat_sessions row not in that set. Model-name-scoped so old 1.5B grades are preserved as separate rows — enables LLM-vs-LLM comparison later.

v1.3.3 Session Notes — 2026-04-05

v1.3.3 — iOS Safari mobile fix fix
User-reported bug: Ask Goose widget on mobile Safari caused zoom-on-type, horizontal scrolling, and crash on return. Root cause: input font-size was 14px (iOS auto-zooms below 16px). Also added overflow-wrap on bubble content, touch-action: manipulation on panel, and overscroll-behavior: contain on messages.
Chatbot grading Google Sheet feature
Created a Google Sheet with per-interaction scoring across 5 dimensions (accuracy, helpful, tone, brevity, fallback). LLM columns pre-filled from existing Ollama grades in chat_grades table. Jose columns left empty for manual grading. Enables LLM vs human score comparison to identify training gaps and feedback opportunities.
sync-goose-sheets.py on Alienware feature
Daily cron job (5:30 AM) reads new Q&A pairs from Supabase chat_messages, looks up LLM scores from chat_grades (populated by grade-ask-goose.py at 3 AM), and appends full untruncated rows to Google Sheet via service account. No redundant Ollama scoring — reuses existing grades. State tracked in ~/.config/goose-sheets-state.json.

v1.3.0 Session Notes — 2026-04-04

v1.3.2 — Mobile chat widget fixes fix
Panel now uses 100dvh + visualViewport API to fit mobile screens and resize when keyboard opens. Swapped weak starter prompt for one with strong RAG grounding.
v1.3.1 — Ask Goose production fix fix
ANTHROPIC_API_KEY was missing from Vercel env. Added fail-fast env validation at module load and SSE error handling in both chat components so errors surface to users instead of infinite spinner.
Ask Goose RAG chatbot built feature
Full RAG chatbot powered by Claude Haiku, Supabase pgvector, and Hugging Face embeddings. 300+ content chunks embedded from 20 writing posts, resume roles, project descriptions, and prompt library entries. Streaming API endpoint with keyword-boosted retrieval (career/tech/project queries force-include relevant chunks) and page-aware context injection (the widget knows which page you're on). Chat bubbles, character counter, 10-question limit with /contact redirect, inline link rendering.
Floating widget on every page feature
AskGooseWidget component renders a chat bubble icon in the bottom-right corner of every page (hidden on /ask-goose where the full-page version lives). Conversations persist across page navigation via sessionStorage. Yellow SVG paw icon in the header for visibility on dark green. Mobile goes full-screen. Widget state (messages, session ID, question count, open/closed) survives navigation.
Keyword-boosted + page-aware retrieval arch
MiniLM-L6-v2 embeddings alone couldn't bridge conversational queries to formal resume descriptions (similarity scores below 0.12). Fix: keyword detection force-includes content types when queries mention career, tech, projects, etc. Page-aware injection: when currentPage is a writing post URL, all chunks from that post are force-included. Chat history trimmed to last 1 exchange to prevent cross-topic bleed.
Chat QA grading pipeline feature
Python script on the Alienware server grades batches of 10 chat sessions using local Ollama (qwen2.5:1.5b — chosen for 6GB RAM constraint on 2011 hardware). Grades on 5 dimensions: accuracy, helpfulness, tone, brevity, fallback handling. Stores results in Supabase chat_grades table, emails report via Resend. Cron job runs daily at 8pm, only triggers when 10+ ungraded sessions exist. First batch scored 3.9/5.

v1.2.0 Session Notes — 2026-04-04

AI content pipeline built feature
Three build-time scripts now enrich every writing post: generate-tags.ts uses Claude Haiku to classify posts into an 11-tag taxonomy, generate-related.ts computes cosine similarity on existing Supabase embeddings to find the 2 most related posts, and reading times are verified via automated word count at 230 wpm. All output is static JSON — zero runtime API calls.
Dynamic tag filtering + post page UI feature
WritingFilter now derives tag pills dynamically from the posts array (was hardcoded to 5 tags, now 11). PostTags component shows tag pills in post headers. RelatedPosts component shows 2 related post cards at the bottom of each article. Both follow the TLDRBadge pattern: client components reading from static data.
New writing post: content pipeline content
"How I Built an AI Content Pipeline for Every Writing Post" — covers tag generation, related posts via vector similarity, the full 7-step publishing pipeline, and how this data feeds into Ask Goose RAG retrieval. Includes terminal visualization and developer sections.

v1.1.0 Session Notes — 2026-04-04

Semantic vector search added feature
Upgraded site search from keyword-only to semantic vector embeddings. pgvector enabled in Supabase, content_embeddings table stores 384-dim vectors (all-MiniLM-L6-v2). Keyword search remains the instant default; vector search fires as fallback when keyword returns 0 results. Queries like "firewall" and "game theory" now surface relevant posts without exact word matches.
HF Inference API for query embedding arch
Initial approach used @xenova/transformers server-side, but ONNX model loading fails in Vercel serverless (timeout + filesystem constraints). Pivoted to Hugging Face Inference API with free-tier token for production query embedding. Local @xenova/transformers still used for build-time content embedding via scripts/generate-embeddings.ts.
New writing post with SVG visualization content
"How I Upgraded Search to Semantic Vector Embeddings" — 600-word case study with recipe card, side-by-side keyword vs. semantic comparison table, inline SVG scatter plot showing content clustering by topic, and Ask Goose (RAG) future feature teaser. Links back to search bar and Linux security posts.

Stack

Framework stack
Next.js 16.1.6, TypeScript, Tailwind CSS 4. Deployed on Vercel via CLI (not GitHub auto-deploy). Local git history is authoritative — never pull from remote.
Database stack
Supabase free tier. Tables: contacts, numerator_rounds, fruit_listings, fruit_requests, fruit_exchanges, email_verifications, market_subscribers. No migration planned — Supabase covers current needs.
Email stack
Resend (free tier, 3k/month). Domain verified for joseandgoose.com. Used by: contact form notifications, Fruit Exchange owner verification, market daily subscriber management.
Writing pipeline stack
Posts defined in app/lib/posts.ts. TL;DR summaries via generate-tldr.ts, topic tags via generate-tags.ts, related posts via generate-related.ts, embeddings via generate-embeddings.ts (all Claude Haiku / MiniLM, build-time). Search index via generate-search-index.ts (prebuild). Each post is a standalone page.tsx.
