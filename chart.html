v1.8.0 Session Notes — 2026-07-23

Passphrase-gated /parking page feature
A page friends can open to see whether there's parking on the block before driving over. The Mac Studio pushes occupancy every 30s to /api/parking/ingest (shared push key) into a single Supabase parking_state row; /parking reads it back through a passphrase-gated /api/parking, with the passphrase sent in a request header rather than the URL. noindex. Only occupancy counts ever leave the house — no video, plates, or faces. Chosen as a push model over a tunnel specifically so nothing on the Studio is network-reachable.
Lazy 2D/3D toggle feature
2D is the instant default at ~120KB; the 3D scene stays completely unloaded until tapped (~24MB of three.js, twin modules, models and textures) so the common case — someone checking spots on cell data — stays fast. Both views are vendored byte-identical from the Sidewalk Watch dashboard at the same absolute paths (/twin3d, /vendor, /assets), so the public page can't drift from the local one. A fetch shim feeds the scene geometry and occupancy from page state and returns an empty /api/live, keeping live vehicle and pedestrian tracks unpublished.
3D ground rendered black — missing HDRI + textures fix
The first 3D deploy showed near-black road and lawn. Cause: lighting.js loads /assets/venice_sunset_1k.hdr as scene.environment and the ground/grass modules pull /assets/textures/road and /grass — the asset copy had only caught .js and .glb, so PBR surfaces had no environment map or albedo. Buildings and hedges use simpler materials and still looked correct, which disguised it as a lighting problem. Source textures were 2048px near-lossless (46MB); resampled to 1024px q80 (4.7MB) since they render on a small embedded canvas.
Vercel git-integration restored; Node 20 pin infra
Deploys had been CLI-only for a reason nobody had noticed: the project was still linked to joseandgoose/starter, a repo since renamed to joseandgoose-site, so the Git link 404'd and Vercel quietly fell back to vercel --prod. Reconnected. Cloud builds then exposed two latent gaps — the project env had never held the full set the app reads (it lived only in the Air's .env.local), and ANTHROPIC_API_KEY / HF_TOKEN were Production-scoped, which breaks Preview builds. Build also pinned to Node 20, since Vercel's new Node 24 image can't compile the old sharp inside @xenova/transformers.

v1.3.4 Session Notes — 2026-04-21

Ask Goose grading: model upgrade to qwen2.5:7b arch
Swapped Ollama model from qwen2.5:1.5b to qwen2.5:7b-instruct-q4_K_M (~Haiku-tier reasoning) for more reliable rubric scoring. 14B variant was pulled but too slow on this CPU (>20min/inference at CPUQuota=250%). RAM upgrade enabled the jump — not CPU — so 7B is the quality ceiling on this machine. Grading takes ~7-8 min/session now vs ~30s on 1.5B.
Cron spread: */15 one-at-a-time arch
Replaced 0 3 * * * daily batch-of-5 with */15 * * * * one-session-per-fire. Script runs with --once flag + lockfile at /tmp/grade-ask-goose.lock, falls through when thermal-gate detects CPU >75°C. Natural ~7-on / 7-off cadence avoids sustained thermal load while catching up ~55 backlogged sessions overnight.
Backfill-aware session picker feature
New fetch_next_ungraded_by_current_model() queries chat_grades filtered by the current OLLAMA_MODEL constant, builds a set of already-graded session IDs, then returns the oldest chat_sessions row not in that set. Model-name-scoped so old 1.5B grades are preserved as separate rows — enables LLM-vs-LLM comparison later.

Keyword-boosted + page-aware retrieval arch
MiniLM-L6-v2 embeddings alone couldn't bridge conversational queries to formal resume descriptions (similarity scores below 0.12). Fix: keyword detection force-includes content types when queries mention career, tech, projects, etc. Page-aware injection: when currentPage is a writing post URL, all chunks from that post are force-included. Chat history trimmed to last 1 exchange to prevent cross-topic bleed.
Chat QA grading pipeline feature
Python script on the Alienware server grades batches of 10 chat sessions using local Ollama (qwen2.5:1.5b — chosen for 6GB RAM constraint on 2011 hardware). Grades on 5 dimensions: accuracy, helpfulness, tone, brevity, fallback handling. Stores results in Supabase chat_grades table, emails report via Resend. Cron job runs daily at 8pm, only triggers when 10+ ungraded sessions exist. First batch scored 3.9/5.

Stack

Framework stack
Next.js 16.1.6, TypeScript, Tailwind CSS 4. Deployed on Vercel via CLI (not GitHub auto-deploy). Local git history is authoritative — never pull from remote.
Database stack
Supabase free tier. Tables: contacts, numerator_rounds, fruit_listings, fruit_requests, fruit_exchanges, email_verifications, market_subscribers. No migration planned — Supabase covers current needs.
Email stack
Resend (free tier, 3k/month). Domain verified for joseandgoose.com. Used by: contact form notifications, Fruit Exchange owner verification, market daily subscriber management.
Writing pipeline stack
Posts defined in app/lib/posts.ts. TL;DR summaries via generate-tldr.ts, topic tags via generate-tags.ts, related posts via generate-related.ts, embeddings via generate-embeddings.ts (all Claude Haiku / MiniLM, build-time). Search index via generate-search-index.ts (prebuild). Each post is a standalone page.tsx.
